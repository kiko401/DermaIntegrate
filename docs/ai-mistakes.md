# ai-mistakes.md — AI 助手历史误判记录

> 记录 AI 助手在本项目中出现的错误推断，用于约束后续会话行为。
> 发现新错误时追加记录，不修改历史条目。

---

## 字段说明

| 字段 | 说明 |
| :--- | :--- |
| 日期 | YYYY-MM-DD |
| 场景 | 当时在做什么 |
| 原错误表述 | 写进文档的原文（引用） |
| 错误类型 | 编造 / 未验证写成已验证 / 目标架构写成现状 / 自相矛盾 / 其他 |
| 依据来源 | 源码 / curl / 对话 / README / 未验证 |
| 正确写法 | 应该写什么 |
| 教训 | 下次触发此场景时要做什么 |

## M-001 — 2026-06-08

| 字段 | 内容 |
| :--- | :--- |
| **场景** | 生成 `docs/project-overview.md` 时列举算法端"已完成"模块 |
| **原错误表述** | `病历 Agent（agents/clinical_agent.py，PAD-UFES-20 schema）`、`病理 Agent（agents/pathology_agent.py，DISEASE_REGISTRY 分诊）`、`整合 Agent（agents/integration_agent.py，RAG 两阶段检索）`，均标注为"已完成" |
| **错误类型** | 目标架构写成现状 + 未验证写成已验证 |
| **依据来源** | README（规划文档）+ 目录树（仅确认文件存在，未读内容） |
| **正确写法** | 拆成两层：①文件已存在（已验证，来自目录树）；②功能描述（推理得出，来自 README，未读源码） |
| **教训** | 目录树有文件名 ≠ 读过该文件内容；README 功能描述 ≠ 代码实现现状。写"已完成"前必须能指向"读源码"或"联调验证"这两种动作之一 |

## M-002 — 2026-06-08

| 字段 | 内容 |
| :--- | :--- |
| **场景** | 写 `docs/known-issues.md` KI-001/002/003 的状态 |
| **原错误表述** | `🟢 已解决（算法端已确认修复，待部署验证）` |
| **错误类型** | 自相矛盾 |
| **依据来源** | 对话（用户说"改完了"）|
| **正确写法** | `🟡 用户口头确认已修改，源码未二次验证（git show），云端部署未确认` |
| **教训** | "已解决"和"待验证"不能同时成立。仅有用户对话确认时，状态上限是 🟡，升为 🟢 需要代码验证或联调验证 |

## M-003 — 2026-06-08

| 字段 | 内容 |
| :--- | :--- |
| **场景** | 写 `CLAUDE.md` 和 `docs/project-overview.md` 中的 AI 服务云端信息 |
