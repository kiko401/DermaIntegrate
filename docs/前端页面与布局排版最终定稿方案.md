DermaIntegrate 前端页面与布局排版最终定稿方案 v1.0
文档说明：本方案基于《用例分析与模块设计 v3.0》，严格对齐 UC-01 至 UC-12 用例与 M1 至 M6 模块，摒弃过度设计，作为前端开发的最终基准蓝图。

一、 路由与页面架构映射 (对应 M1 安全网关层)
路由路径	页面组件	对应用例	核心职责
/login	LoginView.vue	UC-01	凭据输入，httpOnly Cookie 写入
/patients	PatientList.vue	UC-02	医生默认首页，患者检索与 EMPI 映射展示
/clinical/:patient_id	ClinicalView.vue	UC-03, 04, 05, 06, 08	临床多源数据聚合展示 + AI 辅助侧栏实时流式展示
/tasks/:taskId	TaskDetail.vue	UC-05, 06, 07, 08	独立任务详情页（支持实时 Live 模式与历史快照 Snapshot 模式切换）
/admin/users	Users.vue	UC-09	管理员账号 CRUD
/admin/tasks	Tasks.vue	UC-07	管理员 AI 任务监控列表，点击跳转 TaskDetail 快照模式
/admin/integration	Integration.vue	UC-10, UC-11	管理员 EMPI 治理与 SSE 会话治理面板

左侧主视图：调用 GET /api/patients/:id/clinical-view 获取四源数据并按 Tab 展示。某些系统无数据时显示“暂无数据”（UC-03 A1）。
右侧 AI 侧栏：
初始状态：展示该患者最近一次 AI 分析的综合研判内容（如有）。
实时状态 (Live)：若存在 running 任务，建立 EventSource 连接至 GET /api/tasks/:id/stream。
动态渲染：收到 step 事件（image_done / clinical_done / pathology_done）独立并行渲染卡片，缺失字段以“—”占位（UC-05 A1）。收到 heartbeat 忽略。
综合研判覆盖：收到 result 事件，覆盖更新底部区域，展示 risk_level、key_concerns 等。若 status=incomplete，头部显示黄色警告“部分数据缺失，本建议为降级结果”（UC-06 A1）。
断连处理 (UC-08)：组件 beforeUnmount 或路由 beforeRouteLeave 时，执行 eventSource.close()。

三、 核心前端状态管理约束
SSE 生命周期严格闭环：所有包含 EventSource 的组件（ClinicalView 侧栏、TaskDetail Live 模式），必须在 onUnmounted 或 beforeRouteLeave 中执行 close()，确保触发后端 UC-08 资源回收。
动态渲染原则：严禁编写 if(step == 'image_done') 的顺序硬编码。应使用数组或 Map 存储已收到的 step 事件，收到即追加渲染，result 到达即覆盖。
鉴权透明性：前端不存储 Token，所有 API 及 SSE 请求依赖浏览器自动携带 httpOnly Cookie。 Axios 和 EventSource 均需配置 withCredentials: true。
