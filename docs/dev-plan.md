## Phase 0 — 联调准备

**目标**：建立工程端开发环境，验证与 AI 服务的基础连通性，确认 SSE 契约符合 v3.1。

**当前状态**：✅ 完成

### 任务清单

| # | 任务 | 状态 |
| :- | :--- | :--- |
| 0.1 | 建 `feat/app/sse-integration` 分支 | ✅ |
| 0.2 | 初始化 Node/Express 项目骨架（`backend-app/`） | ✅ |
| 0.3 | 实现 `POST /api/tasks/upload`（转发到 AI `/upload`） | ✅ |
| 0.4 | 实现 `GET /api/tasks/:taskId/stream`（流式代理 AI SSE） | ✅ |
| 0.5 | curl 验证 AI `/health` 存活 | ✅ |
| 0.6 | curl/Postman 验证 AI `/upload` 返回 `task_id` | ✅ |
| 0.7 | 用简单 HTML 页面验证 SSE 流事件格式符合 v3.1 | ✅ |

**验收标准**：
- `/health` 返回 `{"status": "UP"}`
- `/upload` 返回包含 `task_id` 的 202 响应
- SSE 流能收到至少一个 `step` 事件和一个 `result` 事件
- `result` 事件中 `status` 为 `"complete"` 或 `"incomplete"`（不是 `"completed"`）
- Node/Express 能透传 SSE 而不缓冲
