> 来源：项目说明文档 + 多模态数据契约 v3.1。
> 状态标记：✅ 已实现 / 🔄 进行中 / ⬜ 待实现

---

## 一、核心问题

| 问题 | 描述 |
|:---|:---|
| 多源数据孤岛 | HIS / PACS / LIS 数据异构，患者标识在各系统不统一，无法形成完整临床视图 |
| 单模态局限 | 初诊常缺病理金标准，单图无法支撑精准分期 |
| AI 不可解释 | 端到端黑盒模型无推理依据，LLM 易产生英文漂移与知识幻觉 |

## 三、非功能需求

| 类型 | 要求 |
|:---|:---|
| 可解释性 | AI 诊断结果必须附带 Grad-CAM 热力图与 RAG Citation，缺失视为推理失败 |
| 渲染隔离 | 前端严禁引入重型 DICOM 解析库，所有影像由后端预处理为 PNG |
| 异步非阻塞 | 应用域严禁同步阻塞调用推理域，必须遵循 SSE 异步契约 |
| 断连回收 | 前端断连必须在 2s 内被感知，推理域随即终止计算，防算力泄漏 |
| 数据隔离 | 推理域 MySQL 3307 与应用域 MySQL 3306 物理隔离，互不访问 |
| 非侵入 | 应用域只读路由获取外部系统数据，严禁对 HIS/PACS/LIS 源系统写入 |
| 契约先行 | 两端协作严格遵循 `docs/api-contract.yaml`，任何接口变更先改契约再改代码 |
